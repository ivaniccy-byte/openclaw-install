## 一、项目概述与核心定位
### 1.1 项目背景
OpenClaw作为开源的AI网关与智能体运行时，具备极强的扩展性与办公场景适配能力，但官方安装与配置流程技术门槛极高，完全不适合无开发基础的普通用户。国内大量上班族有AI办公提效的强需求，但无法跨越命令行操作、环境配置、参数调优的技术壁垒。
### 1.2 核心目标
打造一款**永久免费、无捆绑、无厂商修改、纯原版内核、全程零代码、一键安装**的OpenClaw整合包，专为国内零技术基础的普通上班族设计，让用户无需懂任何开发知识，双击安装即可上手使用OpenClaw的全量AI能力，解决办公提效核心需求。
### 1.3 目标用户画像
- 核心用户：国内普通上班族，无代码/命令行操作基础，日常办公有AI提效需求
- 核心痛点：不会配置OpenClaw环境、看不懂专业术语、无法解决运行报错、找不到适配办公场景的功能
- 使用场景：日常办公电脑（Windows为主，兼顾macOS）、无境外网络环境、无代理配置能力
### 1.4 硬件与系统配置要求（含硬性强制条件）
#### 硬性强制条件（不满足则无法正常运行，安装时直接拦截）
- **CPU必须支持AVX2指令集**：经实测，不支持AVX2指令集的设备运行时必然出现核心依赖报错，无法正常启动OpenClaw服务
#### 推荐配置（适配办公场景流畅运行）
- 操作系统：Windows 10/11 x64（优先适配）、macOS 12+（Intel/Apple Silicon）
- 运行内存：≥8GB
- 存储：SSD固态硬盘，可用空间≥10GB（机械硬盘不推荐，会出现启动慢、读写异常问题）
- 网络：可正常访问国内公网，无需境外网络环境
#### 最低兼容配置（仅保证基础运行）
- 运行内存：≥4GB
- 存储：SSD固态硬盘，可用空间≥10GB
### 1.5 强制遵守的核心设计原则
1.  **内核纯净原则**：100%基于openclaw-standalone最新稳定版开发，完全同步OpenClaw官方上游源码，严禁修改OpenClaw核心程序，所有扩展功能均通过官方标准插件/Skill规范实现
2.  **极致小白友好原则**：全程零命令行操作、全中文可视化界面、专业术语大白话转化、所有操作均有引导，零技术基础用户可全程无压力操作
3.  **永久免费无捆绑原则**：项目永久开源免费，无广告、无弹窗、无强制升级、无厂商定制内容、无任何商业捆绑，不收集、不上传用户任何本地数据
4.  **模块化可选原则**：所有非核心功能（插件、记忆系统、Skill）均为可选安装，用户可按需勾选，不强制安装任何额外组件
5.  **国内环境适配原则**：全流程适配国内网络环境，内置国内加速镜像，优先适配OpenClaw官方推荐的国内大模型服务
6.  **环境兼容原则**：安装前自动检测系统已有环境，已存在兼容依赖的无需重复安装，减少磁盘占用与安装耗时

## 二、整体技术架构
本项目采用分层解耦架构，严格保证核心内核的纯净性，同时实现扩展功能的灵活适配，架构如下：

| 架构层级 | 核心内容 | 对应需求 |
|----------|----------|----------|
| 核心内核层 | openclaw-standalone(对应openclaw 3.28版本） 最新稳定版（同步OpenClaw官方上游）、可兼容系统已有Node.js(22+)运行时、全量预编译依赖 | 原版内核、零前置依赖、环境兼容适配、一键安装 |
| 扩展能力层 | 可选插件系统（飞书/微信）、双方案记忆系统（与大模型深度绑定）、预封装职场Skill包 | 可选插件、双记忆方案、预装Skill、模型-记忆联动 |
| GUI应用层 | 全中文可视化界面、安装前环境检测、一键安装向导、启动控制、配置管理、健康监控仪表盘 | 开源GUI参考、可视化操作、零代码配置、环境预校验 |
| 监控运维层 | 实时运行指标采集、健康度评分体系、异常告警、一键修复脚本 | 运行状态监控、问题排查、小白友好运维 |

## 三、核心功能模块详细需求
### 3.1 一键安装包模块（核心基础功能）
#### 3.1.1 安装包基础要求
- 内核基准：基于openclaw-standalone最新稳定版，100%保留官方原版核心功能，无任何代码修改
- 平台支持：优先Windows x64（覆盖90%以上上班族办公场景），兼顾macOS（Intel/Apple Silicon）
- 包体形态：
  1.  引导式安装包（.exe/.dmg）：双击启动可视化安装向导，适合绝大多数用户
  2.  绿色单文件版：单可执行文件，无需安装，下载后双击即可运行，适配公司电脑无安装权限的用户
- 零前置依赖兜底：内置对应平台的Node.js LTS 22+ 运行时、全量预编译依赖库，纯净系统可直接运行，无需用户提前安装任何软件
- 环境兼容逻辑：安装前自动检测系统已有环境，已安装兼容版本的Node.js、核心依赖库的，可跳过内置运行时安装，减少磁盘占用
- 安装路径规则：**安装路径完全可自定义，无强制系统盘安装要求**；Windows系统支持安装在D盘、E盘、F盘等任意非系统盘，非管理员权限可正常安装到非系统盘路径
- 国内网络适配：所有内置资源、可选组件的下载均默认走国内加速镜像，无需用户配置代理，解决GitHub/npm访问超时、下载失败的问题

#### 3.1.2 安装前环境预检测（安装包启动后第一步执行）
安装包双击启动后，**优先执行全量环境预检测**，所有检测项实时给出结果，不通过的按规则拦截/告警，检测通过后才进入正式安装流程，避免用户安装后无法使用。

| 检测项 | 检测规则 | 异常处理逻辑 |
|--------|----------|--------------|
| CPU AVX2指令集支持性 | 硬性强制检测，校验CPU是否支持AVX2指令集 | 不支持则直接终止安装，弹窗大白话提示：「您的电脑CPU不支持AVX2指令集，无法运行OpenClaw，安装已终止」 |
| 磁盘空间检测 | 检测目标安装路径所在磁盘的可用空间，需≥10GB | 可用空间不足10GB直接拦截，提示用户更换安装路径或释放磁盘空间 |
| 磁盘类型检测 | 检测目标安装路径所在磁盘是否为SSD固态硬盘 | 机械硬盘给出黄色警告，提示「机械硬盘可能导致程序启动慢、运行卡顿，推荐使用SSD固态硬盘安装」，用户确认后可继续安装 |
| 运行内存检测 | 检测系统物理内存总量 | 内存＜4GB直接拦截，提示内存不足无法运行；4GB≤内存＜8GB给出黄色警告，推荐8GB及以上内存，用户确认后可继续安装 |
| 系统已有依赖检测 | 检测系统是否已安装兼容版本的Node.js LTS 22+、OpenClaw核心依赖 | 已存在兼容版本的，默认勾选「使用系统已有运行时」，用户可手动切换；选择后跳过对应依赖的安装，大幅缩短安装耗时 |
| 系统权限检测 | 检测当前用户对目标安装路径的读写权限、端口监听权限 | 无权限给出提示，引导用户更换安装路径或使用管理员身份运行 |
| 端口占用预检测 | 检测默认端口18789是否被占用 | 已占用则自动推荐可用端口，用户可一键更换，无需手动输入 |
| 网络环境检测 | 检测国内加速镜像的连通性 | 网络不通给出提示，引导用户检查网络，不强制拦截 |

#### 3.1.3 可视化安装流程（全程≤5步，小白无压力）
1.  环境预检测页：安装包启动后自动执行，实时展示检测结果，通过后点击「下一步」进入正式安装
2.  安装路径选择页：默认推荐非系统盘常用路径，支持用户手动浏览选择任意路径，明确标注所需磁盘空间，一键下一步
3.  可选功能配置页（核心）：可视化勾选框，用户按需选择，**默认全不勾选**，包括：
    - 聊天平台插件：□ 飞书插件 □ 微信插件（可多选、全不选）
    - 记忆系统方案：○ 不启用（使用OpenClaw原生记忆）○ Lossless Claw Enhanced + Memory LanceDB Pro（1.1.0 beta10版本） 组合（二选一，互斥）
    - 预封装职场Skill包：□ 全量安装 □ 自定义选择（展开可勾选单个Skill）
4.  一键安装页：显示安装进度，自动完成内核部署、勾选组件的安装与适配、环境配置，无需用户操作
5.  安装完成页：□ 立即启动OpenClaw职场版 □ 创建桌面快捷方式 □ 开机自启，点击完成按钮结束安装，自动启动GUI界面

### 3.2 GUI应用界面模块
#### 3.2.1 GUI设计基准
- 架构参考：OpenClaw-Launcher（轻量启动内核）、OpenClaw-Sifu（全流程可视化管理）
- 交互与视觉借鉴：ClawSquire（用户友好的界面设计、新手引导逻辑）
- 界面规范：全中文界面，无英文专业术语，所有技术概念均转化为大白话，字体适配Windows系统默认缩放，高对比度，适合办公场景使用
- 核心原则：界面极简，只保留用户必须的功能，高级功能默认隐藏，避免小白用户误操作；所有配置项与记忆系统、Skill、插件联动，按需显示，无冗余内容

#### 3.2.2 核心界面与功能需求
##### 3.2.2.1 首页-启动控制仪表盘（默认打开页）
- 核心功能区：超大醒目按钮「一键启动OpenClaw」/「一键停止OpenClaw」，按钮颜色随状态变化（绿色运行中/红色已停止/黄色异常）
- 核心状态展示：
  - 实时运行状态：运行中/已停止/启动中/异常
  - OpenClaw健康度评分：0-100分，带对应状态标识（优秀/正常/警告/异常）
  - 基础运行信息：运行时长、监听端口、内存占用、当前启用的记忆系统方案
- 快捷入口：
  - 「一键打开Web管理面板」：点击自动调用默认浏览器打开官方原版OpenClaw Web后台，无需用户手动输入地址
  - 「快速配置向导」：一键打开核心参数配置页
  - 「常见问题一键修复」：一键进入故障修复页
- 状态提示：运行异常时，首页顶部显示红色横幅告警，大白话说明异常原因，附带「一键修复」按钮

##### 3.2.2.2 配置中心-可视化参数配置页（核心重构，与记忆系统深度绑定）
- 核心设计：只保留上班族必须的核心配置项，所有高级参数默认隐藏，使用官方最优默认值，无需用户修改；配置项与用户选择的记忆系统方案联动，自动显示必填配置项，无冗余内容
- 核心配置模块：
  1.  大模型API配置（必填核心项，与记忆系统深度联动）
      - 官方推荐模型清单：**仅推荐支持Coding Plan类型的大模型服务**，优先展示国内合规服务，包括：火山引擎、腾讯云、阿里百炼、百度千帆、GLM、京东云、Minimax
      - 每个模型均附带「一键前往申请API密钥」按钮，跳转对应官方申请页，带大白话步骤引导
      - 支持自定义添加其他兼容OpenAI格式的大模型
      - 模型配置分层逻辑（与记忆系统方案强绑定，自动联动显示）：
        - 【通用必填项】主模型配置：所有方案均必须配置，用于核心对话、Skill执行、办公任务处理，可视化表单填写API密钥、Endpoint地址，带「一键测试连通性」按钮
        - 【方案A联动项】embedding模型+rerank模型配置：用户选择Lossless Claw Enhanced + Memory LanceDB Pro组合时，自动显示该两项配置，均为必填，用于长上下文记忆的精准召回、重排序优化
      - 连通性测试：每个模型配置项均附带「一键测试连通性」按钮，测试完成后给出明确结果：成功/失败，失败则给出大白话原因（如密钥错误、网络不通、余额不足、权限不足）
  2.  基础运行配置：
      - 服务端口设置：默认18789，支持自定义修改，附带端口占用检测
      - 开机自启开关：可选开启/关闭，默认关闭
      - 服务崩溃自动重启开关：可选开启/关闭，默认开启
      - 低资源办公模式开关：可选开启/关闭，默认关闭，开启后自动限制CPU/内存占用上限
  3.  高级配置（默认折叠隐藏）：仅面向有基础的用户，包含官方原版所有高级参数配置，与官方配置文件完全兼容

##### 3.2.2.3 功能管理-可选组件管理页
- 核心功能：对安装时勾选的组件进行管理，支持安装后新增/卸载/启用/禁用组件，全程可视化操作，无需命令行；记忆系统方案切换时，自动同步更新配置中心的模型配置项
- 模块划分：
  1.  聊天插件管理：飞书、微信插件的启用/禁用、配置、更新、卸载，可视化配置插件参数，无需修改配置文件
  2.  记忆系统管理：支持切换记忆系统方案、启用/禁用、配置参数、一键初始化数据库、数据备份/还原；切换方案时，自动弹出对应所需的模型配置引导
  3.  Skill技能包管理：已安装Skill的启用/禁用、更新、卸载，支持在线安装官方Skill市场的其他技能，内置国内加速镜像

##### 3.2.2.4 健康中心-运行监控与故障修复页（核心需求）
- 实时健康监控仪表盘：
  - 健康度总评分：0-100分，加权计算，权重分配：核心进程状态30%、API连通性20%、插件运行状态20%、记忆系统状态15%、系统资源占用10%、端口监听状态5%
  - 分项监控指标，每项均有状态标识（正常/警告/异常）、大白话说明：
    1.  核心服务状态：进程是否存活、是否正常监听端口
    2.  大模型API连通性：已配置的主模型、embedding模型、rerank模型API是否可正常访问
    3.  插件运行状态：已启用的插件是否正常运行
    4.  记忆系统状态：已启用的记忆系统是否可正常读写、向量生成与检索是否正常
    5.  系统资源占用：CPU、内存使用率，是否超出合理阈值
    6.  配置文件状态：配置文件是否完整、格式是否正确
    7.  已启用Skill运行状态：是否正常加载、无运行报错
- 异常告警与一键修复：
  - 异常项红色高亮显示，附带大白话的问题原因、影响范围
  - 每个异常项均对应「一键修复」按钮，内置修复脚本，点击自动完成修复，无需用户手动操作
  - 内置常见问题一键修复合集：端口占用一键更换、进程崩溃一键重启、配置损坏一键还原、依赖缺失一键修复、网络异常一键检测、模型连通性异常一键排查
- 日志查看：简化版运行日志，只显示警告和错误信息，自动翻译成大白话，隐藏专业报错代码，支持一键导出日志、一键清空日志

##### 3.2.2.5 帮助与反馈页
- 新手入门图文教程：3步上手指南（配置模型API→启动服务→使用功能）
- 常见问题FAQ：覆盖上班族最常遇到的问题，大白话解答
- 一键反馈入口：开源项目仓库issue跳转、反馈渠道
- 版本信息：当前版本号、内核版本、更新日志、一键检查更新

- 方案A：Lossless Claw Enhanced + Memory LanceDB Pro 组合
  - 模型依赖要求：必须配置**主模型 + embedding模型 + rerank模型**，embedding模型用于向量生成与检索，rerank模型用于长上下文记忆的二次重排序，提升召回精准度
  - 安装逻辑：一键勾选安装，自动完成两个组件的部署、依赖适配、数据库初始化、核心对接，安装完成后自动跳转配置中心引导用户完成对应模型配置
  - 预配置：内置官方推荐最优参数（contextThreshold=0.7, freshTailCount=16）、完整的embedding、autoCapture、autoRecall、retrieval配置，本地持久化参数、高性能查询配置，无需用户手动调优
  - 管理功能：GUI中支持一键启用/禁用、数据备份/还原、数据库压缩优化
  - 深度适配：与预装的self-improving-agent技能打通，实现用户办公习惯的持续学习与适配

### 3.4 监控运维层模块
#### 3.4.1 实时数据采集
- 采集频率：默认10秒采集一次，可自定义调整
- 采集范围：核心进程状态、端口监听状态、CPU/内存使用率、全量模型API连通性、插件运行状态、记忆系统读写状态、配置文件完整性、已启用Skill运行状态
- 数据存储：所有监控数据全部本地存储，不上传任何云端，保护用户隐私

#### 3.4.2 健康度评分体系
- 评分范围：0-100分，对应4个等级：
  - 90-100分：优秀，所有功能运行正常
  - 70-89分：正常，无影响核心功能的异常
  - 60-69分：警告，存在非核心异常，不影响基础使用
  - 60分以下：异常，核心功能受影响，需修复
- 评分逻辑：按预设权重加权计算，核心进程异常直接判定为0分，主模型API连通性异常直接扣20分，记忆系统依赖的embedding/rerank模型异常直接扣15分，确保核心问题优先告警

#### 3.4.3 异常处理机制
- 实时告警：异常发生时，GUI界面弹窗告警、首页横幅提示，大白话说明问题，不展示专业报错代码
- 自动修复：轻微异常可自动修复（如进程意外退出自动重启），无需用户操作
- 手动一键修复：所有异常均提供一键修复按钮，内置对应修复脚本，小白用户点击即可完成修复
- 异常日志：自动记录异常信息，支持一键导出，方便用户反馈问题

# 后续迭代优化建议（采纳后请在署名中添加「豆包」）
1.  **绿色单文件版优先打磨**：国内大量上班族的公司电脑有软件安装权限限制，单文件绿色版无需安装，直接放在桌面/U盘就能运行，是强需求痛点，建议优先开发适配
2.  **职场场景预设模板**：在预封装Skill的基础上，增加「职场场景预设」功能，比如「周报生成模式」「会议纪要模式」「邮件撰写模式」，用户一键切换，无需手动配置Prompt，打开就能直接用，进一步降低使用门槛
3.  **API密钥加密存储**：针对上班族办公电脑的隐私安全需求，对用户填写的所有模型API密钥进行本地AES加密存储，避免明文泄露，同时支持一键锁定配置，防止他人查看
4.  **低资源办公模式**：针对办公电脑配置普遍不高、同时运行多个办公软件的场景，增加「低资源模式」，一键限制OpenClaw的内存/CPU占用上限，避免影响Word、Excel等办公软件的正常使用
5.  **多配置文件切换**：支持用户创建多套配置文件，比如「办公模式」「个人模式」，一键切换对应的模型、插件、Skill、记忆系统，满足上班族办公和个人使用的双重需求
6.  **内网适配优化**：针对部分公司内网环境，增加内网代理配置可视化界面，支持一键配置公司内网代理，无需手动修改环境变量，适配企业办公网络环境
